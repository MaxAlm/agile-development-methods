Analys

Analys

När jag började med det här projektet hade jag aldrig innan jobbat med varken HTML, CSS eller JavaScript. Jag vet inte riktigt vad jag hade för förväntningar inför kursen då arbetssättet inom webbutveckling och webbdesign var ett helt okänt område för mig. Under kursens första lektion lärde vi oss om grunderna inom webbutveckling. Vi lärde oss om webbutvecklingens olika byggstenar, hur dem används, samt dem absoluta grunderna för hur man bygger och designar sin egen hemsida. Vi fick sedan i uppgift att klara ett antal uppgifter på en sida som heter W3Schools, en sida där man kan öva och lära sig om allt inom webbutveckling. Uppgifterna i W3Schools var till en viss hjälp, men jag känner ändå att jag lärde mig som mest när jag satt och provade mig fram på egen hand. Att dessutom kolla upp hur man implementerar olika element och funktioner vid behov kändes väldigt lärorikt.

När jag väl hade blivit någorlunda bekväm med arbetssättet inom HTML och CSS så började jag bygga på min hemsida. Med hjälp av Azure DevOps var det enkelt att lägga upp en tidsplan, samt hålla koll på vad som har gjorts och vad som ska göras. Under arbetet med det första utkastet till min hemsida byggde jag bara på utan någon vidare plan, detta ledde sedermera till att jag stötte på en hel del oväntade problem. Förutom att strukturen till hemsidan blev rörig och att CSS-filerna fylldes med onödiga klasser, så resulterade det hela i att jag fick börja om från noll. Anledningen till detta var att jag helt hade glömt att hemsidan i slutändan skulle vara fullt responsiv. Detta betyder att webbsidan måste vara fullt tillgänglig på både surfplattor, mobil och dator utan att användaren ska behöva zooma in för att kunna förstå och läsa sidans innehåll. Detta var inget jag hade haft i åtanke under skapandet av min hemsida och kom därför fram till att jag skulle spara mer tid på att helt enkelt börja om från början.

För att undvika föregående misstag så var jag mer noggrann och försiktig vid uppbygget av min andra webbsida. Jag höll mig till en stilren svartvit design med en gråvit gardient som bakgrund för att undvika att sidans innehåll skulle smälta in allt för mycket med bakgrunden. Jag såg också till att varje ändring samt ny implementation var fullt responsiv innan jag gick vidare, här var jag också noga med att inte lägga till allt för mycket extra ordinärt eller överflöd för att underlätta processen. Jag såg också till att återanvända så mycket kod som möjligt i CSS. Istället för att, till exempel, göra en ny CSS-fil för menyn på varje sida, så använde jag istället samma CSS-fil igen. Förutom detta kommenterade jag även all min kod noggrant. Dels för att göra det enklare för utomstående att förstå koden, men också för att underlätta för mig själv vid möjliga ändringar sent in i projektets gång.

Under projektets gång har jag varit noga med att uppdatera och följa min tidsplan på Azure DevOps. Det har underlättat en hel del då ett projekt av denna storlek annars lätt kan kännas överväldigande och rörigt, något som undveks med hjälp av DevOps Backlog och Sprint funktion. Att dessutom kunna gå in och se hur man låg till tidsmässigt via DevOps analysfunktion var till stor hjälp. Förutom DevOps har jag också använt mig av Git under arbetet med hemsidan. Med Git var det enkelt att göra säkerhetskopior, något som underlättade när man skulle göra stora och riskfyllda ändringar i projektet. Jag tycker att arbetet med hemsidan har varit en väldigt rolig och lärorik uppgift. Man har fått experimentera och prova sig fram en hel del, och jag ser fram emot att jobba mer med HTML, CSS och JavaScript i framtiden.
